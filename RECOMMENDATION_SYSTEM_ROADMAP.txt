# Homestead Recommendation System: 8-Week Implementation Roadmap

This document outlines a step-by-step journey to implement a hybrid recommendation system for the Homestead platform using Machine Learning (Python).

---

## Phase 1: Foundations & Data Preparation

### Week 1: Understanding Recommendation Systems & Data Gathering
**Goal:** Learn the theory and identify/extract data from the current application.
*   **Concepts:** Content-Based Filtering, Collaborative Filtering (User-based vs. Item-based), Matrix Factorization (SVD), Hybrid Systems.
*   **Actionable Steps:**
    1. Set up a Python environment (venv) and install Jupyter Notebook.
    2. Identify data in MongoDB/PostgreSQL: User IDs, Property attributes (location, price, amenities), and Interaction data (Reviews, Bookings).
    3. Create a Node.js export script to generate `users.csv`, `properties.csv`, and `interactions.csv`.
*   **Resources:**
    - Google ML Crash Course: https://developers.google.com/machine-learning/recommendation
    - Coursera Recommender Systems (Audit for free): https://www.coursera.org/specializations/recommender-systems

### Week 2: Data Preprocessing & Exploratory Data Analysis (EDA)
**Goal:** Clean data and understand patterns using Python and Pandas.
*   **Concepts:** Pandas data manipulation, handling missing values, encoding categorical data, Interaction Matrices.
*   **Actionable Steps:**
    1. Load CSVs into Jupyter using Pandas.
    2. Analyze rating distributions and booking frequency.
    3. Create a User-Item Interaction Matrix.
*   **Resources:**
    - Kaggle Pandas Tutorial: https://www.kaggle.com/learn/pandas
    - EDA for Recommender Systems: https://towardsdatascience.com/exploratory-data-analysis-for-recommender-systems-99238318b871

---

## Phase 2: Building the Models

### Week 3: Content-Based Filtering (CBF)
**Goal:** Recommend properties based on their attributes (features).
*   **Concepts:** Vectorization (TF-IDF), Cosine Similarity.
*   **Actionable Steps:**
    1. Combine property features (description, location, amenities) into text strings.
    2. Apply `TfidfVectorizer` from scikit-learn.
    3. Calculate Cosine Similarity matrix.
    4. Function: `recommend_similar_properties(property_id)`.
*   **Resources:**
    - Scikit-learn TF-IDF: https://scikit-learn.org/stable/modules/generated/sklearn.feature_extraction.text.TfidfVectorizer.html
    - Datacamp CBF Tutorial: https://www.datacamp.com/tutorial/recommender-systems-python

### Week 4: Collaborative Filtering (CF)
**Goal:** Recommend properties based on user behavior patterns.
*   **Concepts:** Memory-based CF (KNN), Model-based CF (SVD).
*   **Actionable Steps:**
    1. Install `Surprise` Python library.
    2. Load interaction data into a Surprise Dataset.
    3. Train an SVD algorithm.
    4. Function: `predict_user_rating(user_id, property_id)`.
*   **Resources:**
    - Surprise Library Docs: https://surpriselib.com/
    - Real Python CF Guide: https://realpython.com/build-recommendation-engine-collaborative-filtering/

### Week 5: Building a Hybrid System
**Goal:** Combine CBF and CF to solve the "Cold Start" problem.
*   **Concepts:** Weighted Hybrids, Cold Start strategies.
*   **Actionable Steps:**
    1. Normalize scores from CBF and CF models.
    2. Create a master function `get_hybrid_recommendations(user_id)`.
    3. Implement fallback to "Popular Properties" for brand new users.

## Phase 3: Evaluation, Deployment, and Integration

### Week 6: Model Evaluation & Tuning
**Goal:** Quantify model performance using data science metrics.
*   **Concepts:** Train/Test splits, RMSE, Precision@K, Recall@K.
*   **Actionable Steps:**
    1. Split historical data (80% train / 20% test).
    2. Evaluate SVD model using RMSE.
    3. Hyperparameter tuning using Cross-Validation.
*   **Resources:**
    - Evaluation Metrics: https://towardsdatascience.com/evaluation-metrics-for-recommender-systems-df56c6611093

### Week 7: Serving the Model via a Microservice
**Goal:** Wrap the Python model in an API for the Node.js backend to consume.
*   **Concepts:** FastAPI/Flask, Model Serialization (Joblib/Pickle).
*   **Actionable Steps:**
    1. Save trained models and TF-IDF matrices using `joblib`.
    2. Create a FastAPI app with endpoints for user recommendations and similar properties.
    3. Test locally via Postman.
*   **Resources:**
    - FastAPI Tutorial: https://fastapi.tiangolo.com/tutorial/
    - Deploying ML Models: https://towardsdatascience.com/deploying-a-machine-learning-model-as-a-rest-api-4a03b865c166

### Week 8: Integration with Homestead
**Goal:** Connect the Node.js/React stack to the Python Recommendation API.
*   **Actionable Steps:**
    1. **Node.js:** Update controllers to fetch data from the FastAPI service using `axios`.
    2. **React:** Build `<RecommendedProperties />` and `<SimilarListings />` components.
    3. **Pipeline:** Schedule a daily task (CRON) to export new interactions and retrain the model.

 1. How to Gather More Properties (The "Item" Data)
  Since you can't wait for hundreds of real hosts to sign up, you should "seed" your
  database with realistic data.

   * Option A: Open Datasets (Recommended for ML)
      The best resource for this is Inside Airbnb
  (http://insideairbnb.com/get-the-data.html). They provide massive CSV files of real
  property listings (including descriptions, prices, amenities, and reviews) for cities
  all over the world.
       * Action: Download a dataset for a city (e.g., New York or London), write a
         simple Python or Node.js script to parse the CSV, and bulk-insert them into
         your MongoDB/Postgres database.
   * Option B: Synthetic Data Generation
      Use a library like Faker.js (https://fakerjs.dev/) in your backend. You can write
  a script that generates 500 properties with random addresses, prices between $50-$500,
  and random combinations of amenities (WiFi, Pool, Kitchen).
   * Option C: Rapid Manual Entry
      Use a tool like Postman (https://www.postman.com/) to send 20-30 POST requests to
  your /api/properties endpoint with data copied from real rental sites to at least have
  a diverse "starter pack."

  ---

  2. How to Gather User Interaction Data (The "Interaction" Data)
  To build a recommendation system, you need to know what users like. You gather this in
  two ways:

  A. Explicit Feedback (What they tell you)
  This is already partially in your schema.
   * Ratings & Reviews: When a user stays at a property, they leave a 1-5 star rating.
   * Action: You need to encourage users to review. For your ML development, you can
     "fake" these reviews by randomly assigning ratings from "fake users" to your
     "seeded properties."

  B. Implicit Feedback (What they do — Much more powerful)
  Most users don't leave reviews, but they do browse. You need to track these "events":
   1. Property Views: Every time a user clicks a property card to see the details.
   2. Wishlists/Favorites: When a user clicks a "Heart" icon.
   3. Search Queries: What locations and price ranges are they searching for?

  How to implement this technically (Future Task):
  You would create a new table/collection called Interactions or ActivityLogs.

   1 {
   2   "userId": "60a...",
   3   "propertyId": "71b...",
   4   "interactionType": "view", // or "wishlist", "click"
   5   "timestamp": "2023-10-27T10:00:00Z",
   6   "duration": 45 // seconds spent on page
   7 }
  Every time a user visits a Property Detail page in React, your frontend would send a
  "ping" to the backend to log this view.

  ---

  3. Immediate Strategy: Synthetic Interaction Data
  Since you don't have thousands of real users yet, you cannot train a Collaborative
  Filtering model. To learn the ML process now, you should generate a Synthetic
  Interaction Dataset.

  The Process:
   1. Seed Properties: Get 100 properties (using Inside Airbnb or Faker).
   2. Seed Users: Create 50 fake users.
   3. Create "Personalities" for Fake Users:
       * User A likes "Cheap apartments in the city."
       * User B likes "Luxury villas with pools."
   4. Generate "Fake Interactions": Write a Python script that makes User A "view" and
      "rate highly" properties that match their personality.
   5. Train your Model: Use this fake data to train your SVD model.

  Why do this?
  It allows you to build the entire pipeline (Data -> Model -> API -> React) and verify
  it works. When real users eventually start visiting your site, you simply swap the
  "fake data" with "real data" and retrain the model.

  Summary of your "Data Gathering" Week 1-2:
   1. Download an "Inside Airbnb" CSV.
   2. Write a script to import 100-200 of those listings into your Homestead database.
   3. Learn Python/Pandas to create a script that generates a CSV of 1,000 "fake"
      interactions between 50 users and those 200 properties.
   4. Now you have enough data to actually start the ML training in Week 3!
